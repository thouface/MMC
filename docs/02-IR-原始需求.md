# MMC — 功能模块与用户故事

## 1. 用户画像

| 角色 | 典型场景 | 核心诉求 |
|------|----------|----------|
| 数码爱好者 | 拥有多台手机/平板/笔记本 | 互通便捷、操作流畅 |
| 家庭管理者 | 跨品牌智能家居设备 | 统一控制、联动场景 |
| 内容创作者 | 手机、相机、电脑之间传素材 | 大文件高速传输 |

## 2. 功能模块清单

### M1 设备发现与配对（MVP）
- F101 局域网自动发现（mDNS / NSD / Bonjour）
- F102 扫码配对（备用）
- F103 手动配对（输入 IP + 端口）
- F104 设备信息展示（名称、类型、系统、IP）

### M2 设备管理（MVP 基础）
- F201 设备列表（区分在线/离线）
- F202 状态刷新（心跳）
- F203 设备重命名 / 删除

### M3 远程控制（MVP）
- F301 屏幕镜像
- F302 触控远程操作（Android 优先）
- F303 控制权限申请弹窗

### M4 文件传输（MVP）
- F401 文件选择（图片/视频/文档）
- F402 分块发送（最大 2 GB）
- F403 接收确认
- F404 传输进度（速度/剩余时间）

## 3. 核心用户故事

1. **作为新用户，** 我打开 App 后能自动扫描附近设备，选中后一键配对。
2. **作为发送方，** 我能在设备详情页选择文件并开始传输，实时看到进度。
3. **作为接收方，** 我能看到发送请求弹窗，选择接受或拒绝。
4. **作为控制端，** 我能镜像目标设备屏幕并进行基本触控操作。
5. **作为长期用户，** 我能查看已配对设备列表，管理命名与状态。

## 4. 非功能性需求

- 兼容性：Android 8.0+、iOS 13.0+
- 安全：设备配对需双向确认；传输数据加密
- 可维护性：模块化架构，便于后续扩展协议与设备类型
- 性能：镜像延迟 ≤ 200 ms，传输速率 ≥ 10 MB/s
