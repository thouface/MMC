# MMC — 产品愿景与初始需求

## 1. 问题背景

随着个人拥有的智能设备数量不断增加（手机、平板、笔记本、智能家居等），
用户面临以下痛点：

- 品牌生态割裂，跨品牌设备之间难以互通
- 需要在多个 App / 系统界面之间切换操作
- 文件传输流程繁琐（微信/邮件/云盘，受限于大小与网速）
- 缺乏统一、轻量、隐私友好的多设备管理体验

## 2. 愿景

**让一台手机控制身边所有智能设备，在不同设备之间自由传输文件。**

- 打开即用：无需注册登录，无需繁琐设置
- 隐私优先：数据仅在局域网内传输，不上云
- 跨平台：一套代码覆盖 Android / iOS，后续扩展 PC

## 3. 成功标准（MVP）

- 用户在 30 秒内完成两台同局域网设备的发现与配对
- 文件传输速度 ≥ 10 MB/s（局域网千兆环境）
- 屏幕镜像延迟 ≤ 200 ms
- 应用冷启动时间 ≤ 3 秒
- Android 端远程触控操作可用
