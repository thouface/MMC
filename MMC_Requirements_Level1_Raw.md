# MMC — 需求拆解：原始需求层

---

## 原始需求（Raw Requirements）

> 直接从用户和业务背景中提取的原始陈述，未经加工和分类。

### 1. 问题与动机

| # | 原始陈述 |
|---|----------|
| R-001 | 用户拥有多台智能设备（手机、平板、笔记本、智能家居等），品牌不同 |
| R-002 | 不同品牌设备之间难以互通 |
| R-003 | 用户需要在多个 App / 系统界面之间切换操作 |
| R-004 | 文件传输流程繁琐（依赖微信/邮件/云盘，受限于大小与网速） |
| R-005 | 用户缺乏统一、轻量、隐私友好的多设备管理体验 |

### 2. 产品愿景

| # | 原始陈述 |
|---|----------|
| R-006 | 让一台手机控制身边所有智能设备 |
| R-007 | 在不同设备之间自由传输文件 |
| R-008 | 打开即用：无需注册登录 |
| R-009 | 打开即用：无需繁琐设置 |
| R-010 | 隐私优先：数据仅在局域网内传输 |
| R-011 | 隐私优先：数据不上云 |
| R-012 | 跨平台：一套代码覆盖 Android / iOS |
| R-013 | 跨平台：后续扩展 PC |

### 3. 功能期望

| # | 原始陈述 |
|---|----------|
| R-014 | 设备发现：用户在 30 秒内完成两台同局域网设备的发现与配对 |
| R-015 | 文件传输：速度 ≥ 10 MB/s（局域网千兆环境） |
| R-016 | 屏幕镜像：延迟 ≤ 200 ms |
| R-017 | 应用冷启动时间 ≤ 3 秒 |
| R-018 | Android 端远程触控操作可用 |

### 4. 使用约束

| # | 原始陈述 |
|---|----------|
| R-019 | 不需要互联网连接 |
| R-020 | 不需要云服务器中转 |
| R-021 | 不需要用户注册账号 |
| R-022 | 不需要复杂的配置流程 |

### 5. 质量属性（隐含）

| # | 原始陈述 |
|---|----------|
| R-023 | 操作简单，学习成本低 |
| R-024 | 传输过程安全，不被第三方窃取 |
| R-025 | 设备发现快速，无需等待 |
| R-026 | 传输稳定，中断后能恢复 |
